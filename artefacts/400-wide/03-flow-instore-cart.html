  Note over Gateway: cart contains 3 line items — enrich each with per-store availability

  loop For each cart item (SKU, requestedQty)
    Cart->>Inventory: getStock(storeId, sku)
    Note right of Cart: RESP GET · p95 ≤ 3ms per lookup
    alt Cache hit — shelf stock found (items 1 and 3)
      Inventory-->>Cart: { sku: "LMP-100", qtyOnShelf: 4, sellThroughToday: 8, lastSync: "10:29 UTC" }
      Note left of Inventory: p95 ≤ 3ms
      Note over Cart: status ← "available" · no SAP call needed
    else Cache miss — no batch data for this SKU at this store (item 2: TRL-X9-42)
      Inventory-->>Cart: null
      Note left of Inventory: p95 ≤ 3ms · key absent — falling back to SAP ECC inline
      Cart->>SAP: checkStockRealtime(storeId, sku)
      Note right of Cart: IDoc synchronous lookup · bypasses 15–30 min batch cycle
      Note left of SAP: p95 ≤ 400ms · WAN + ERP query overhead
      alt SAP confirms stock on hand
        SAP-->>Cart: { sku: "TRL-X9-42", qtyOnHand: 2, warehouseBin: "B-12", lastPhysicalCount: "09:15 UTC" }
        Note left of SAP: p95 ≤ 400ms
        Note over Cart: status ← "available (SAP-confirmed)"
+400ms penalty vs cache hit
      else SAP cannot confirm — source-of-truth has no record
        SAP-->>Cart: { sku: "TRL-X9-42", status: "not-found", reason: "no-stock-at-location" }
        Note left of SAP: p95 ≤ 400ms
        Note over Cart: status ← "unavailable"
confirmed by SAP source-of-truth
+400ms penalty vs cache hit
      end
    end
  end
